# 副本罐子分配器｜逐輪操作規格（更新版）

## 目標
實作一個純前端工具（HTML + CSS + JavaScript，單檔 `index.html`），
讓玩家在副本進行中，依序輸入每一輪出現的罐子，
系統立即回覆：

- 第1隊站哪裡
- 第2隊站哪裡
- 第3隊站哪裡

---

## 使用情境（UX）
玩家在實戰中是「逐輪」知道資訊，不是一次知道 3 輪。

流程如下：

1. 目前輪次顯示「第 N 輪」
2. 玩家按按鈕選擇該輪出現的 3 個屬性罐
3. 按「確認本輪並給建議」
4. 系統將建議結果寫入表格（1隊/2隊/3隊 + 王吸收）
5. 鎖定本輪結果，進入下一輪
6. 共進行 3 輪

補充按鈕：

- 回上一輪（修正誤輸入）
- 全部重置

---

## 遊戲機制

### 每輪罐子
每輪會出現：

- 五屬性中隨機 3 種：會心 / 氣血 / 移速 / 攻速 / 範圍
- 固定 1 個狂暴

所以每輪共 4 罐。

### 小隊
共有 3 隊（第1隊、第2隊、第3隊），每輪各站 1 罐。

因此：

- 3 個罐子被小隊站走
- 1 個罐子給王吸收

---

## 核心限制

### 1. 屬性不可重複（跨輪）
對同一隊來說，五屬性（會心/氣血/移速/攻速/範圍）在 3 輪中不能重複站。

- 例如：第1隊第1輪站過會心，則第2、3輪不能再站會心。

### 2. 狂暴可重複
狂暴不受上述限制，同一隊可在多輪站狂暴。

### 3. 逐輪鎖定
每輪確認後即鎖定，下一輪建議會承接前面已站過屬性。

### 4. 攻速+範圍警告
若同一輪同時出現「攻速」與「範圍」，按下確認時顯示紅字警告：

- 請自斷重新開始

並中止該輪計算。

---

## 每輪建議的最佳化順序
在「該輪所有合法站法」中選最佳：

1. 優先讓玩家站到機制簡單的罐子（狂暴 > 會心 > 氣血 > 移速 > 攻速 > 範圍）
2. 第一優先相同時，優先讓第3隊吃狂暴
3. 前兩項相同時，保留較高彈性（各隊剩餘可用屬性空間較大）

---

## 輸入方式（更新）
不再使用三輪下拉一次輸入。

改為「按鈕點擊」：

- 顯示 5 個屬性按鈕
- 顯示 2 個 Debuff 小按鈕（第1隊禁狂暴 / 第2隊禁狂暴）
- 每輪必須點選剛好 3 個，且不可重複
- 狂暴由系統自動加入

Debuff 小按鈕說明：

- 預設為關閉（通常不需要按）
- 若開啟「第1隊禁狂暴」，本輪第1隊不可被分配到狂暴
- 若開啟「第2隊禁狂暴」，本輪第2隊不可被分配到狂暴
- Debuff 僅本輪有效，確認本輪後會自動清空，下一輪需重新點選

---

## 輸出內容（更新）

### 1. 已鎖定輪次表格
欄位：

- 輪次
- 出現罐子
- 第1隊
- 第2隊
- 第3隊
- 王吸收

表格需固定顯示第1~第3輪，並用 Highlight 標示最新建議那一行。

### 2. 累積統計
- 每隊已站過屬性

### 3. 無解提示
若某輪在既有鎖定條件下（含 Debuff 限制）無合法站法，顯示錯誤訊息。

---

## 技術要求

- 使用：HTML / CSS / JavaScript
- 不可使用：後端 / 資料庫 / API
- 交付：單一 `index.html`
